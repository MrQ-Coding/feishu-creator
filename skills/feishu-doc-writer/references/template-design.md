# 方案设计模板

## 1. 推荐章节骨架

1. `一、背景与目标`
2. `二、现状分析`
3. `三、方案设计`
4. `四、方案对比`（存在多个候选方案时）
5. `五、实施计划`
6. `六、风险与应对`
7. `七、总结`

## 2. 各章节写作要点

### 背景与目标

- 用 2-3 句话描述问题。
- 用编号列表列出具体目标。
- 区分必须实现和可选实现。

### 现状分析

- 简要描述当前架构或工作流。
- 突出本方案要解决的痛点。
- 架构较复杂时配架构图（Graphviz，`rankdir=LR`，subgraph 分组各层）。

### 方案设计

- 第一句话给出核心思路。
- 按组件或层次拆分子章节。
- 每个组件说明：做什么 → 怎么做 → 为什么这样做。
- 按组件拆分时配架构图（Graphviz，`rankdir=LR`，subgraph cluster 分组），展示组件间依赖和数据流。
- 涉及多服务/多模块的请求-响应交互时配时序图（PlantUML，`A -> B: msg` 语法），展示调用顺序。
- 领域模型/实体关系设计配类图（PlantUML），展示属性、方法和关联关系。
- 工单/订单等生命周期设计配状态机图（PlantUML），展示状态流转和触发条件。

### 方案对比

使用对比表格：

| 维度 | 方案 A | 方案 B |
|------|--------|--------|
| 复杂度 | ... | ... |
| 性能 | ... | ... |
| 可维护性 | ... | ... |

表格后给出明确推荐和理由。

### 实施计划

- 按阶段划分，每个阶段有具体交付物。
- 顺序步骤用有序列表。
- 标注阶段间的依赖关系。

### 风险与应对

使用「风险 + 对策」配对：

- **风险**: 描述具体风险
  **应对**: 具体缓解措施

## 3. 注意事项

- 先给结论再展开分析。
- 图表聚焦——一张图说明一个概念。
- 尽量量化（延迟、成本、工作量）。
- 事实与观点明确区分。
