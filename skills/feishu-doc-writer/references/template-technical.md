# 技术分析模板

## 1. 推荐章节骨架

1. `一、概览`
2. `二、概念介绍`
3. `三、源码入口`
4. `四、源码解析`
5. `五、关键子流程`
6. `六、总结`

## 2. 章节写作模式

每节默认按以下节奏推进：

1. 概述行（`概览：...`）
2. 主路径说明（有序步骤）
3. 证据（精简代码片段、命令输出或日志）
4. 衔接句（`这一步最终会进入...`）

## 3. 源码片段格式

路径：

```text
frameworks/base/services/core/java/com/android/server/wm/RootWindowContainer.java
```

片段：

```java
// frameworks/base/services/core/java/com/android/server/wm/RootWindowContainer.java
void startHomeOnAllDisplays(...) {
    // 只保留关键行
}
```

证明：

```text
这段逻辑证明主流程会先遍历显示设备，随后再调用目标启动入口方法。
```

## 4. 对比章节格式

使用「术语 + 简短解释」列表：

- **同步流程**: 调用链在当前线程内连续推进，便于直接定位下一跳。
- **异步流程**: 通过消息或回调转交执行，需要额外确认触发点与时序。

## 5. 测试/验证报告格式

骨架：

1. `一、测试结论`
2. `二、验证范围`
3. `三、验证方式`
4. `四、效率记录`
5. `五、关键问题`
6. `六、建议`

`三、验证方式` 应标注证据来源：

- MCP 工具调用
- 本地服务 / 脚本执行
- 代码阅读推理

每条问题按以下结构组织：

1. 现象（`现象：...`）
2. 证据（`证据：返回结果 / 截图 / 关键字段`）
3. 推断（`推断：...`，视情况添加）
4. 建议（`建议：...`）

## 6. 图表指引

以下场景应配图，图表紧跟对应正文段落之后：

| 章节 | 触发条件 | 推荐图表 |
|------|----------|----------|
| 源码入口 | 涉及多个类/模块的入口调用 | 调用关系图（`create_graphviz_diagram_block`） |
| 源码解析 | 主流程涉及 3 步以上 | 流程图（`create_graphviz_diagram_block`） |
| 关键子流程 | 异步回调、跨线程/进程交互 | 时序图（`create_plantuml_diagram_block`） |
| 概念介绍 | 3 个以上组件的层级关系 | 架构图（`create_graphviz_diagram_block`） |

## 7. 注意事项

- 先写主路径，再展开分支。
- 片段精简，不要粘贴整个文件。
- 行内代码仅用于标识符和源码路径，不用于普通中文词汇。
